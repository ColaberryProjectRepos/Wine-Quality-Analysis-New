Step 2 - Transforming the Data in Power Query Editor:


  • In Power BI, change the default font of the project to "DIN." This can be accomplished by going to View > Customize Current Theme > Text, setting all text to "DIN," Font Size to 12, and Font Color to Navy Blue (#12239E). 
  • NOTE: This can be changed as needed later on in the Format pane based on specific visuals, but setting it as default is used for most visuals in the Power BI report, which helps to save time.




  • Since this data is clean, it did not require much tweaking, except for capitalizing the column names and abbreviating some of them, such as 'Free Sulfur Dioxide' to 'Free SO2' and 'Total Sulfur Dioxide' to 'Total SO2', which would be helpful later on for shorter visual names.
  •  Some columns were rounded and converted to percentages as per the business requirements, which is an important step to ensure their functional use as visuals in the report later.



  • Adding a Date/Time custom column that renamed  later "(Last Refreshed)" in report card visual.  There are many ways to do this but this is the way we will be doing it. To add the column go to Power Query Data and Add Column > Custom Column. Put this as the DAX formula:  =DateTime.LocalNow()






  •  Creating a calculated column for " (Volume Tested Liters)" Volume Tested (Liters) = RAND() * (1000 - 500) + 500. This calculation generates a random volume between 500 and 1000 liters, simulating the volume of wine tested for each sample or observation in the dataset.
