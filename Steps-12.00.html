Step 3 - Adding Additional Tables:


Measure Table: Go to the "Home" tab and click "Enter Data" and just rename the blank table to "Measures Table" and click "OK."

Create your measures in your measure table. Then, follow this method for the rest of the measures. This approach helps you organize each measure based on the data you want to access, such as percentage increase and arrow color for the indicator, matrix table for the segment, and waterfall table for your waterfall chart reports later on.







  • Repeat this steps for "(% + Arrow Color), (Matrix Table) and ( Waterfall %) "measure Tables.


1. Click your "Measures Table" and click on "New Measure" at the top. Here we will make our main Alcohol measure. In this measure, you will type the following DAX code:

 Avg Alcohol OG = 
AVERAGEX(
    ALL('Wine-Quality'), 
    'Wine-Quality'[Alcohol]
)

2. Now open another new measure to make our next measure which shows the Average Alcohol . In this measure, you will type the following DAX code: 

Avg Alcohol New = 
AVERAGEX(
    ALLSELECTED('Wine-Quality'),
    'Wine-Quality'[Alcohol])






  • The difference in the result is that "Avg Alcohol OG" gives a consistent average for the entire dataset, while "Avg Alcohol New" provides an average that can change based on user selection and interactions within the report or visual filters.
  • Repeat step 1 & 2 for Fixed Acidity, Volatile Acidity, Chlorides, Citric acid, Free SO2, Residual sugar,  Sulphate and Total SO2 columns.



The % + Arrow Color Table: The "(% + Alcohol)" measure serves as the correct sub-KPI for the main KPI. It calculates changes only when there's a non-zero difference between the new and original values; if not, it leaves the result blank, thus tracking variations in wine properties across different periods effectively. 

1. In this measure, you will type the following DAX code: 
% + Alcohol = 
VAR Original = [Avg Alcohol OG]
VAR New = [Avg Alcohol New]
VAR Increase = DIVIDE(New - Original, Original)
RETURN IF(
    NOT ISBLANK(New) && Increase <> 0, 
    (Increase),
    BLANK()
)
Meanwhile, the "(Arrow Alcohol)" measure determines the direction of an arrow, either upward or downward, based on the sub-KPI's result. 

2.  In this measure, you will type the following DAX code:
Arrow Alcohol = 
    IF(
        [Avg Alcohol New] < [Avg Alcohol OG],
         UNICHAR(9660),    // Unicode for downward arrow
         UNICHAR(9650)     // Unicode for upward arrow 
    ) 

 "(Alcohol IND Color)" changes to reflect an increase or decrease when comparing new to original values. 

3.  In this measure, you will type the following DAX code:
Alcohol IND Color = 
    IF(
        [Avg Alcohol New] > [Avg Alcohol OG],
          "green", "red"
    )

  • Repeat this steps for  Fixed Acidity, Volatile Acidity, Chlorides, Citric acid, Free SO2, Residual sugar,  Sulphate and Total SO2 columns with replaced names for the values. Just simply copy and paste the DAX codes and replace them with the corresponding values.




The Matrix Table: calculating minimum and maximum alcohol levels and their respective scaled values. The use of MIN, MAX, and DIVIDE functions facilitates the segmentation of wines based on alcohol content for in-depth analysis or reporting.
