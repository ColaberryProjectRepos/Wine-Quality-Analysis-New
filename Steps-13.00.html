Step 4 - Adding Calculated Columns and Measures:


Creating a calculated column for Alcohol, Fixed Acid, Residual Sugar, FSO2 and TSO2 within the "Wine-Quality" table.





  • In this calculated column for Alcohol DAX code: "(Alc_Segment)" = SWITCH(
        TRUE(),
        'Wine-Quality'[Alcohol] >= 8 && 'Wine-Quality'[Alcohol] < 9.39, "8 - 9.38",
        'Wine-Quality'[Alcohol] >= 9.39 && 'Wine-Quality'[Alcohol] < 10.78, "9.39 -       10.77",
        'Wine-Quality'[Alcohol] >= 10.78 && 'Wine-Quality'[Alcohol] < 12.17, "10.78 - 12.16",
        'Wine-Quality'[Alcohol] >= 12.17 && 'Wine-Quality'[Alcohol] < 13.56, "12.17 - 13.55",
        'Wine-Quality'[Alcohol] >= 13.56 && 'Wine-Quality'[Alcohol] <= 14.90, "13.56 - 14.90",
        BLANK()
    )

  • The formula utilizes the SWITCH function to categorize wines by their alcohol content. For instance, "Alc_Segment" the formula assigns labels such as "8 - 9.38" or "9.39 - 10.77" to different alcohol percentage ranges, making this column valuable for segmenting wines in reports or visualizations according to their alcohol content.

To create a wine quality waterfall chart in Power BI without having all necessary columns, disconnected table with "(Waterfall %)" name was created.



  • The  "(Chemical Rate)"  SWITCH measure was created on  "(Waterfall %)" table that dynamically represents the values based on the selected measure from the disconnected "Wine-Quality" table. This approach allows you to populate the axis and value fields of the waterfall chart, showing the incremental contributions of different quality attributes, such as acidity or sugar content, to the overall wine quality.
  • In this table  "(Waterfall %)" type the following DAX code measure:
Chemical Rate =
SWITCH (
    SELECTEDVALUE ( 'Waterfall %'[ID] ),
    1, [% + Fixed],
    2, [% + Volatile Acidity],
    3, [% + Citric],
    4, [% + Residual Sugar],
    5, [% + Chloride],
    6, [% + Free SO2],
    7, [% + Total SO2],
    8, [% + Sulphate],
    9, [% + Alcohol])
